===========================================
   1. VUE D'ENSEMBLE
===========================================

L'API Catalog permet d'accéder et de modifier les données de produits, 
catégories, marques, fournisseurs, partenaires et autres informations du catalogue.

URL DE BASE : https://cifbetjefyfocafanlhv.supabase.co/functions/v1/catalog-api/v1/{companySlug}

FORMAT DES RÉPONSES : JSON
MÉTHODES SUPPORTÉES : GET, PATCH, POST, OPTIONS
PAGINATION : Supportée
FILTRES : Disponibles selon l'endpoint

===========================================
   2. AUTHENTIFICATION
===========================================

AUTHENTIFICATION PAR CLÉ API
- Header requis : x-api-key
- Valeur : Clé API générée dans l'interface d'administration
- Validation : La clé doit être active et associée à l'entreprise

EXEMPLE DE REQUÊTE :
```
GET /catalog-api/v1/{companySlug}/products
Headers:
  x-api-key: your_api_key_here
  Content-Type: application/json
```

PERMISSIONS PAR CLÉ API :
- products: Lecture des produits
- products_write: Écriture sur les produits
- categories: Accès aux catégories  
- brands: Accès aux marques
- packs: Accès aux packs de produits
- environmental: Données environnementales
- suppliers: Lecture des fournisseurs
- suppliers_write: Écriture sur les fournisseurs

NOTE : Les endpoints partenaires (GET /partners, GET /partners/{slug})
sont publics et ne nécessitent pas de clé API.

===========================================
   3. SÉCURITÉ ET FILTRAGE
===========================================

FILTRAGE AUTOMATIQUE DES PRODUITS ADMIN :
- L'API publique filtre automatiquement les produits marqués avec `admin_only: true`
- Seuls les produits avec `admin_only: false` ou `admin_only: null` sont exposés

ISOLATION DES PACKS PARTENAIRES :
- Les packs assignés à un ou plusieurs partenaires via `partner_packs` sont
  automatiquement exclus de l'endpoint GET /packs
- Ces packs restent accessibles uniquement via GET /partners/{slug}
- Cette isolation garantit l'exclusivité des offres partenaires

PERMISSIONS D'ÉCRITURE :
- Les endpoints PATCH requièrent la permission `products_write`
- Les endpoints fournisseurs requièrent la permission `suppliers`

## 7.3 PACKS PARTENAIRES - OPTIONS DE PERSONNALISATION

Les packs assignes a un partenaire peuvent avoir des options de
personnalisation permettant au client final de choisir des produits
supplementaires dans des categories definies.

L'API enrichit automatiquement chaque option avec les details complets
des produits autorises (nom, image, prix, variants).

STRUCTURE D'UNE OPTION :

| Champ | Type | Description |
|-------|------|-------------|
| id | uuid | Identifiant unique de l'option |
| category_name | string | Nom de la categorie (ex: "Tablettes") |
| is_required | boolean | L'option est-elle obligatoire ? |
| max_quantity | number | Quantite maximale selectionnable |
| allowed_product_ids | uuid[] | IDs de variantes (product_variant_prices) autorisees |
| allowed_products | object[] | Produits autorises avec details complets |

⚠️ NOTE IMPORTANTE sur allowed_product_ids :
- Ces IDs sont des **variant price IDs** (UUIDs de la table `product_variant_prices`), PAS des product IDs directs.
- L'API effectue un filtrage intelligent : seuls les produits ayant au moins une variante selectionnee apparaissent dans `allowed_products`.
- Les `product_variant_prices` retournes pour chaque produit sont limites aux seules variantes presentes dans `allowed_product_ids`.
- Un produit avec 0 variante selectionnee est automatiquement exclu de la reponse.
| position | number | Ordre d'affichage |

STRUCTURE D'UN PRODUIT AUTORISE (allowed_products[]) :

| Champ | Type | Description |
|-------|------|-------------|
| id | uuid | ID du produit |
| name | string | Nom du produit |
| slug | string | Slug du produit |
| image_url | string | URL de l'image |
| price | number | Prix d'achat |
| monthly_price | number | Prix mensuel |
| brand_name | string | Nom de la marque |
| category_name | string | Nom de la categorie |
| short_description | string | Description courte |
| product_variant_prices | object[] | Variants avec prix |

EXEMPLE DE REPONSE (option enrichie) :

## 7.4 PRESTATAIRES EXTERNES

Les prestataires externes sont des fournisseurs de services tiers
(téléphonie, assurance, etc.) dont les produits sont présentés sur
la page partenaire sous forme de cartes de services.

Le client peut souscrire directement via le site du prestataire
(lien externe). Les produits prestataires ont une période de
facturation : `monthly`, `yearly` ou `one_time`.

## 7.5 ISOLATION DU CATALOGUE

Comportement automatique :
- Un pack assigné à au moins un partenaire via `partner_packs`
  est automatiquement EXCLU du endpoint GET /packs
- Ce pack reste visible dans l'admin (back-office)
- Ce pack est accessible via GET /partners/{slug}
- Si le pack est retiré de tous les partenaires, il réapparaît
  dans GET /packs

## 7.6 GÉNÉRATION DE NUMÉRO DE DOSSIER SÉQUENTIEL

```
POST /catalog-api/v1/{companySlug}/partners/{partnerSlug}/next-reference
```

AUTHENTIFICATION : Clé API requise (x-api-key)

Génère et réserve le prochain numéro de dossier séquentiel au format
`ITC-{année}-OFF-{numéro}`. Le numéro est atomique et garanti unique.

Appelez cet endpoint AVANT de soumettre une demande via
`create-product-request` pour obtenir le `reference_number` à inclure
dans le payload.

RÉPONSE :
```json
{
  "reference_number": "ITC-2026-OFF-9976"
}
```

FLUX D'UTILISATION :

1. Appeler POST /partners/{slug}/next-reference
   → Recevoir { "reference_number": "ITC-2026-OFF-9976" }

2. Appeler POST /create-product-request avec le champ
   "reference_number": "ITC-2026-OFF-9976" dans le payload

3. L'offre est créée dans Leazr avec ce numéro de dossier

NOTE : Si `reference_number` n'est pas fourni dans create-product-request,
un numéro séquentiel est généré automatiquement côté serveur.

===========================================
   9. GESTION DES ERREURS
===========================================

## CODES D'ERREUR HTTP

- **400 Bad Request** : Requête malformée
- **401 Unauthorized** : Clé API manquante ou invalide
- **403 Forbidden** : Permission insuffisante
- **404 Not Found** : Endpoint ou ressource introuvable
- **429 Too Many Requests** : Limite de débit atteinte
- **500 Internal Server Error** : Erreur serveur

## EXEMPLES D'ERREURS

### Permission insuffisante (403)
```json
{
  "error": "Permission denied: products_write required"
}
```

### Partenaire introuvable (404)
```json
{
  "error": "Partner not found"
}
```

### Champ requis manquant
```json
{
  "error": "purchase_price is required"
}
```

## RÉCUPÉRER LA PAGE PARTENAIRE (JavaScript)
```javascript
const API_BASE = 'https://cifbetjefyfocafanlhv.supabase.co/functions/v1/catalog-api/v1';
const COMPANY_SLUG = 'your-company-slug';

async function getPartnerPage(partnerSlug) {
  const response = await fetch(
    `${API_BASE}/${COMPANY_SLUG}/partners/${partnerSlug}`
  );
  
  if (!response.ok) {
    throw new Error('Partenaire non trouvé');
  }
  
  const data = await response.json();
  // data.partner : infos du partenaire
  // data.packs : packs exclusifs avec options
  // data.providers : prestataires avec produits
  return data;
}
```
